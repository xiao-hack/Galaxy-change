# 🚀 Update / 更新日志

## 2026-05-26

#### 🚀 优化

> **实时扫描展示：右侧参数面板（Box 2）更新逻辑重构**
>
> - **打破卡顿冰封**：彻底重构了 `galaxySql` 漏洞扫描中参数面板（Table 2）的渲染机制。以往由于数据更新只在单次扫描完全结束后触发，导致扫描期间右侧参数表长时间空白或卡死。
> - **毫秒级实时刷新**：现已打通后台扫描线程与前台 EDT（事件分发线程）的实时联动。在扫描过程中，一旦有新的 Payload 测试完毕，系统会自动判定并实时追加到右侧参数列表中展示。无需等待扫描结束，即可像泉水流淌般实时掌握扫描细节与漏洞探测进展。

#### 🛡️ 修复

> **并发安全护航：彻底攻克 GalaxySql 流量多线程错乱与锁死缺陷**
>
> - **终结参数串包**：解决了原 `GalaxySqlPanel` 在高并发流量扫描下“接口与参数内容对不上，参数会混乱显示到下一个接口”的严重 Bug。原因为临时变量 `temp_data` 和 `original_data_len` 声明为了共享的类成员变量，导致并发请求相互覆盖。已将这些变量**局部变量化**，彻底隔绝线程间的数据冲突。
> - **强健的线程同步**：对不安全的容器（`ArrayList`）读写以及清空（Clear）操作加锁保护（`synchronized`），并对 `conut` 计数器的增量和表格行插入进行了原子化同步，彻底规避了高并发下的 `ConcurrentModificationException`。
> - **Swing 线程合规**：将所有 UI 渲染刷新以及选行行为，统一通过 `SwingUtilities.invokeLater` 派发至事件分发线程（EDT）执行，确保了界面的绝对稳定。
>
> **逻辑完美闭环：修复数字扫描开关（is_int）默认未对齐问题**
>
> - **消除状态偏差**：修复了“值是数字则进行-1、-0”复选框在界面显示为“未勾选”，但后台逻辑却依然强行对数字参数执行注入探测的逻辑缺陷。通过将后台变量 `is_int` 的默认初始值修正为 `0`，完美与 UI 初始状态对齐，实现通过界面勾选开关 100% 精准控制注入行为。

## 2026-05-18

#### ✨ 增加

> **解密后注入扫描：新增 galaxySql 漏洞扫描面板**
>
> - **突破痛点**：针对加密传输环境下的 SQL 注入难以利用传统漏扫（如 Sqlmap）进行检测的问题，新增 **galaxySql** 面板（基于 XiaSql 深度优化定制开发）。该功能可自动拦截解密后的请求包，利用插件自身的加解密引擎对测试 Payload 进行二次加密发送，实现对加密 HTTP 流量的无缝 SQL 注入主动测试。
> - **丰富扫描配置**：
>   - **数字型自适应**：支持对数字参数自适应执行 `-1`/`-0` 逻辑差分比对。
>   - **自定义 Payload**：内置可视化编辑并支持持久化存储自定义 payload（`xia_SQL_diy_payload.ini`），支持空格自动 URL 编码及参数值置空模式。
>   - **Cookie 扫描**：支持一键开关 Cookie 字段的注入探测。
>   - **多域名白名单**：支持以逗号分隔的多域名白名单过滤，保障扫描目标的精准与安全。
> - **多维检测逻辑**：智能分析响应长度差值及响应延时（如 sleep 盲注检测），结合状态码判定并高亮标记（`✔`）疑似漏洞。
> - **极致交互体验**：设计了清晰的双分栏联动表格，并嵌入 Burp 原生 Request/Response 视图，测试参数、Payload 细节、响应耗时、长度变化等核心数据一目了然，方便快速定位及复现漏洞。

#### 🛡️ 修复 

> **高并发稳定性：修复大量请求下流量捕获异常**
>
> - **修复消息丢失**：将请求上下文缓存 `hookedIds` 升级为线程安全的 `ConcurrentHashMap.newKeySet()`，彻底解决了并发大量请求时，由于原 `HashSet` 的读写冲突导致的流量漏抓与解密失败问题，极大提升了应对高并发请求的稳定性。

## 2026-04-14

#### ✨ 增加

> **RSA 算法：原生支持多块（Multi-block）加解密**
>
> - **功能突破**：彻底解决了 `too much data for RSA block` 报错。插件现已能够自动处理超过 128 字节（1024位密钥）或 256 字节（2048位密钥）的超长 RSA 密文。
> - **兼容性**：保留原有 API 签名，现有脚本无需修改即可直接获得分块加解密能力。
>
> **脚本引擎优化：跨语言参数透明化**
>
> - **类型自动转换**：将 `CryptoUtil` 中加解密接口的入参统一升级为 `Object` 类型。
> - **解决 ProxyArray 报错**：通过内部 `toJavaBytes` 逻辑，使得 GraalJS (JavaScript) 或 Python 脚本可以直接传递字节数组，免去了手动处理复杂类型包装的繁琐，极大降低了脚本编写门槛。
>
> **自动补全增强**：优化了编辑器逻辑，输入 `CodeUtil.` 或 `CryptoUtil.` 后能更顺滑地触发方法提示。
>
> **界面美观**：增加了超长字符串自动换行，避免更换密钥操作麻烦

#### 🛡️ 修复 

> **系统稳定性：修复关键 NullPointerException (NPE)**
>
> - **响应包加密修复**：解决了在 Burp Proxy 模块中，由于响应包发往客户端阶段（`hook_response_to_client`）请求对象（Request）可能为空，从而导致的插件崩溃问题。
> - **防御性编程**：在 `Request.of(HttpRequest)` 入口处增加了 Null 检查。即便 Burp 的请求上下文丢失，插件也会安全返回 `null` 而不是抛出 `toByteArray()` 错误，显著提升了在高并发或特殊流量下的鲁棒性。
>
> - **SM2 国密算法：格式自适应增强**
>   - **Raw 格式强制修正**：针对 128 字节的原始坐标数据增加了强制判定逻辑。即便首字节巧合为 `0x04`，只要长度符合 128 字节，代码都会正确补齐前缀并解析，解决了 1/256 概率下的解密失败问题。
>   - **多格式兼容**：支持 Raw (128B)、Standard (129B) 以及 ASN.1 (DER) 格式的自动识别与互转。
>   - **坐标对齐**：修复了 `BigInteger` 转换时偶发的字节偏移问题，确保坐标始终严格对齐为 32 字节。
